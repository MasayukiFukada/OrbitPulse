## ✅ 実装済みの機能 (Progress)

### 1. プロダクトバックログ管理
- **CRUD操作**: バックログアイテム(PBI)の登録・編集・削除。
- **Whyの重視**: なぜ作るのかを必須項目として管理。
- **受け入れ条件**: アイテムごとの完了定義を複数行で保持。
- **見積もり**: フィボナッチ数列によるストーリーポイント設定。

### 2. スプリント管理
- **期間設定**: 開始日・終了日の管理。
- **ステータス管理**: Planning -> Active -> Completed の遷移。
- **キャパシティ管理**: 
    - 日ごとの標準パルス数(4 Pulse = 2時間)を自動生成。
    - 日ごとの微調整と「体調メモ」の保存。

### 3. タスク・実績管理 (Planning Board)
- **PBIタスク**: PBIに紐付く具体的な作業の切り出し。
- **スプリント共有ToDo**: 特定のPBIに属さない雑務（期限付き）の管理。
- **Pulse計測**: 見積Pulse(Est)と実績Pulse(Act)のポチポチ入力。
- **インプレース編集**: タスク名のクリック編集。
- **リアルタイム集計**: スプリント全体の「予定 vs 実績」をダッシュボードに表示。

### 4. アーキテクチャ・基盤
- **クリーンアーキテクチャ**: Domain / Application / Infrastructure / Presentation の分離。
- **データベース**: SQLite + Drizzle ORM による堅実な永続化。
- **共通UI**: サイドバーナビゲーションの導入。

## 🚀 今後の予定 (Future Plans)

### 短期的な目標
- **ダッシュボードの可視化**:
    - バーンダウンチャート（日ごとの残りPulseの推移）。
    - 予定 vs 実績の予実管理グラフ。
- **ToDoバックログ画面**:
    - まだどのスプリントにも入れてへんToDoタスクを管理・ストックする場所の作成。

### 中長期的な目標
- **ゲーミフィケーション**:
    - 完了したストーリーポイントやPulseを「XP（経験値）」として集計。
    - ユーザーレベルや称号の仕組み。
- **Google カレンダー連携**:
    - 予定がある日のキャパシティを自動で減らす。
- **AIスクラムマスター (MCP)**:
    - 過去の実績から「今スプリント、ちょっと詰め込みすぎちゃう？」とアドバイスをくれる機能。

### UI/UX ブラッシュアップ
- **Optimistic Updates**: 保存待ちを感じさせないサクサクした操作感。
- **デザインの洗練**: 「宇宙・軌道」をイメージした、見てるだけでワクワクするデザイン。
